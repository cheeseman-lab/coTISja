# scripts

This directory contains files with helper functions, python processing scripts, and SLURM scripts

## Processing scripts

- `filter_ribotish.py`: Filters ribotish results down to a set of high-confidence TISs. Briefly, the procedure is:
    1. Identify all transcripts represented in the TIS data which have strong annotations: MANE_Select or Transcript support level 1/2/3
    2. Filter to TISs mapped to only the selected transcript IDs, which have a minimum normalized readcount of 0.1 RPM and which have minimum significance values of 0.05 for the combined Harr enrichment and in-frame tests
    3. Keep the canonical start site
    4. Iteratively remove all TISs within 30 nt downstream of any previously kept TIS, then select the next highest readcount TIS to keep.
    - Input files are sourced and outputs are saved according to the manifest files (`ribotish_replicate_manifest.csv`, `ribotish_sample_manifest.csv`)
- `fit_leaky_scanning_model.py`: Fits a generalized linear model representing the mechanistic factors involved in leaky ribosome scanning from the 5' UTR. Specifically the model predicts the translation efficiency at the canonical start site using the number of upstream TISs, the similarity of the canonical sequence context to the Kozak consensus sequence, and the length of the 5' UTR.
    - Depends on: the compiled TIS tables from `notebooks/tis_gene_analysis.ipynb` and the QC-masked readcount matrices from `scripts/run_differential_analysis.py`
    - Outputs: The formatted input data per transcript with predictors as columns (`glm_model_inputs`), a summary of parameter estimates and model fit (`canonical_translation_model_summary.csv`), the estimated covariance between predictors (`canonical_translation_covariance_matrix.csv`), and a table of annotations of TISs per transcript. Files are saved to `data/translation_models`
- `run_differential_analysis.py`: Runs differential TIS translation and gene expression analysis for all pairwise comparisons between samples using DESeq2
    - Depends on: the formatted `tisdiff` inputs from `filter_ribotish.ipynb`
    - Outputs: Reformatted input metadata and matrices prepared for DESeq2 (`input_sample_metadata.csv`, replicate-level `input_counts_matrix.csv`, condition-level `rna_summed_replicate_counts.csv` and `rpf_summed_replicate_counts.csv`), tables of summary statistics from DESeq for each pairwise comparison (`pairwise_wald`, `pairwise_wald_rna`), fitted translation efficiency values, with (`translation_efficiency_vst_matrix`) and without (`log_translation_efficiency_coeffs.csv`) a variance stabilizing transformation, and results from a likelihood ratio test for whether there is any difference accross samples for a given TIS (`lrt_results.csv`). Results are saved to `data/tisdiff_results`

### python

- `run_ribotish_tisdiff_array.sh`: Runs an arrayed SLURM job to export count matrices for each pairwise comparison between samples.
    - Inputs are sourced from the manifest file generated by `notebooks/filter_ribotish.ipynb` (`ribotish_tisdiff_manifest.tsv`) and count matrices in the format of \[TIS x replicate-level Ribo-seq and RNA-seq samples\] are exported to `data/tisdiff_results`

## Helper functions

- `analysis_pipeline_helpers.py`: Contains functions for majors steps or data munging in the processing scripts above, used primarily in `fit_leaky_scanning_model.py` and `run_differential_analysis.py`
- `filter_utils.py`: Contains functions for manipulating and reformatting tables ot TIS data, used primarily in `filter_ribotish.py`, `notebooks/tis_gene_analysis.ipynb`, `notebooks_sample_comparison.ipynb`
- `plotting.py`: Contains various plotting functions used in analysis. Plot types include volcano plots, geneset enrichment plots, forest plots, and most notably a customized barplot arranged along a visualization of a transcript body. This latter function `plot_tis_over_cell_lines()` may facilitate quick or standardized visualization of TIS translation in the context of nearby sites, as a quick pythonic alternative to IGV. I might recommend a variant of this fucntion for an interactive Dash app.
